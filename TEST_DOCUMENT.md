1、引言

1.1 编写目的
简要介绍本次测试文档编写的目的，针对运维工单系统进行环境部署和功能测试。

1.2 测试目标
确保前端、后端及移动端项目能够在本地顺利构建，核心功能基本可运行。

1.3 测试环境
- 操作系统：Ubuntu 24.04 LTS
- Node.js 20.19.2
- pnpm 9.15.4
- Java 21.0.7
- Maven 3.8.7
- Gradle 8.14（通过 Gradle Wrapper 自动下载）
- Docker 未安装（部署脚本依赖 Docker，实际测试时未运行）

1.4 测试人员
测试由课程设计小组成员完成。

1.5 参考材料
项目仓库中的 `README.md`、`FINAL_TESTING_REPORT.md` 等文档。

2、测试用例设计
2.1 测试数据准备
使用项目提供的默认数据或脚本，不额外构造数据。

2.2 测试策略
- **静态检查**：确保依赖安装无报错。
- **构建测试**：尝试使用 Maven、pnpm、Gradle 分别构建后端、前端和移动端。
- **运行测试**：受限于环境（无 Docker），未实际启动完整服务，仅验证构建步骤。

2.3 测试原则
- 依赖尽量本地化，减少外网下载失败的影响。
- 构建过程若出现不可解决的网络问题，以日志记录告警。

2.4 测试用例设计
| 编号 | 模块 | 测试内容 | 预期结果 | 实际结果 |
|----|------|---------|---------|---------|
| TC1 | 后端 | 执行 `mvn test` | 依赖下载成功，若有测试用例则执行 | 网络限制导致依赖下载失败 |
| TC2 | 前端 | `pnpm install && pnpm run build` | 构建完成生成 `dist` 目录 | 构建成功 |
| TC3 | Android | `./gradlew assemble` | 成功生成 APK 文件 | 需要下载大量依赖，因时间限制未完成 |

3、测试过程
1. 使用 `apt-get` 安装 Maven，成功安装并验证版本。
2. 在 `ops-workorder-simple-gh/backend` 目录运行 `mvn test`，因无法访问 `repo.maven.apache.org` 导致构建失败。
3. 切换至 `frontend-web`，执行 `pnpm install` 与 `pnpm run build`，构建顺利完成并产出静态文件。
4. 尝试进入 `frontend-web/android` 运行 `./gradlew assemble`，Gradle Wrapper 会自动下载 Gradle，后续构建过程因依赖下载耗时较长终止。

4、结论
- 本地环境已安装 Maven，前端项目能够成功构建。
- 后端与移动端构建受网络及依赖下载限制，需配置离线仓库或提前准备依赖。
- 建议在具备 Docker 与网络访问条件的环境下再次完整部署和测试。
